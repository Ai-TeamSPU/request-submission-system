# AI Rules

Before editing any file

Read every document inside /docs.

Never ignore these rules.

## Never

- Change Business Logic
- Change API
- Change Database
- Change Authentication
- Change Authorization
- Rename existing functions
- Remove features

## Allowed

- UI
- UX
- Animation
- Accessibility
- Responsive Design

If functionality may change

STOP

Explain why first.

# Critical Rule

This project is already functional.

Preserving existing functionality has the highest priority.

Visual improvements must NEVER change the application's behavior.

If a UI improvement requires changing business logic, APIs, database structure, routing, authentication, or application flow:

STOP.

Explain the reason.

Wait for explicit user approval before continuing.

Every completed task must maintain 100% backward compatibility.
