## 3. ผลการจับคู่: ดึงได้ 9 คอลัมน์ จาก 16 คอลัมน์

### ✅ ดึงได้โดยตรง (9 คอลัมน์)

| คอลัมน์ Excel | → | คอลัมน์ `requests` | หมายเหตุ |
|---|---|---|---|
| **ชื่ออาจารย์** | → | `teacher_name` | ตรง |
| **คณะ** | → | `faculty` | ตรง |
| **สาขา** | → | `department` | ตรง (ถ้าเป็น `-` เก็บว่าง) |
| **รหัสวิชา** | → | `course_code` | ตรง |
| **ชื่อวิชา** | → | `course_name` | ตรง |
| **กลุ่ม** | → | `section` | แปลงเป็น string |
| **วันที่สอน** | → | `date` | แปลง DD/MM/YYYY → YYYY-MM-DD |
| **เวลาเรียน** | → | `time_range` | ตรง |
| **ห้องเรียน** | → | `classroom` | ตรง |

### 🔍 ดึงได้ทางอ้อม (1 คอลัมน์)

| คอลัมน์ `requests` | วิธีได้มา |
|---|---|
| `email` | ใช้ **ชื่ออาจารย์** จาก Excel ไป match กับ `faculty.name_th` / `faculty.full_name_th` → ดึง `faculty.email` ออกมา |

### ⚙️ ระบบสร้างให้อัตโนมัติ (4 คอลัมน์)

| คอลัมน์ `requests` | ค่าที่กำหนด |
|---|---|
| `id` | สร้างอัตโนมัติ เช่น `REQ-20260615-00001` |
| `problem_type` | ค่าเริ่มต้น: `"ไม่ได้เช็คอิน"` |
| `reason` | ค่าเริ่มต้น: `"นำเข้าจากรายงานอาจารย์ไม่เช็คอิน"` |
| `submitted_date` | เวลาปัจจุบัน `now()` |

### ➖ ไม่มีข้อมูล (2 คอลัมน์)

| คอลัมน์ `requests` | ค่า |
|---|---|
| `attachment_name` | `''` (ว่าง) |
| `attachment_data` | `null` |

## 4. คอลัมน์ Excel ที่ใช้ไม่ได้ (7 คอลัมน์)

| คอลัมน์ Excel | เหตุผล |
|---|---|
| ลำดับ | เป็นแค่ running number |
| รหัสบุคลากร | ไม่มีคอลัมน์ใน `requests` |
| ประเภท | ไม่มีคอลัมน์ใน `requests` |
| เวลาเช็คอิน | เป็นข้อมูลการเช็คอินจริง ไม่เกี่ยวกับคำร้อง |
| Scan หน้าเข้างาน | ข้อมูลเสริม |
| QR ห้องเรียน | สถานะ QR code |
| นศ. เช็คอิน | จำนวนนักศึกษาที่เช็คอิน |
