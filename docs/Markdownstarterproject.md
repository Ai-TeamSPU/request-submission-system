# Concept & Function ของระบบคำร้องขอเช็คอินย้อนหลัง (RSMS)

ระบบจัดการการขอเช็คอินย้อนหลังสำหรับมหาวิทยาลัย ประกอบด้วยผู้ใช้งาน 5 บทบาท:

## บทบาทผู้ใช้งาน (User Roles)

| บทบาท | สิทธิ์การใช้งาน |
|--------|----------------|
| **อาจารย์** (`teacher`) | ยื่นคำร้อง, ดูคำร้องของตัวเอง |
| **คณบดี** (`dean`) | อนุมัติ/ปฏิเสธคำร้อง, ดูประวัติ |
| **ผอ.สำนักงานวิชาการ** (`director`) | ดูคำร้องที่อนุมัติแล้ว, ตั้งค่าอีเมลแจ้งเตือน |
| **วิชาการ** (`academic`) | ดูคำร้องที่อนุมัติแล้ว, ตั้งค่าอีเมลแจ้งเตือน |
| **ผู้ดูแลระบบ** (`admin`) | เข้าถึงทุกเมนู + จัดการข้อมูลหลัก |

## การทำงานหลัก

- **Login** ด้วย email อย่างเดียว ไม่มี password — ผู้ใช้ใหม่จะได้สิทธิ์ `teacher` เป็นค่าเริ่มต้น
- **อาจารย์** กรอกฟอร์มยื่นคำร้อง → ข้อมูลบันทึกลง Supabase → ระบบส่งอีเมลแจ้งคณบดีอัตโนมัติ
- **คณบดี** พิจารณาอนุมัติ/ปฏิเสธ → ระบบส่งอีเมลแจ้งอาจารย์เจ้าของคำร้อง
- **Admin** จัดการบทบาทผู้ใช้ ดูสถิติ Dashboard (Pie Chart) นำเข้า Excel

## ข้อมูลในฟอร์มคำร้อง

- ชื่อผู้สอน, คณะ/วิทยาลัย, สาขา
- รหัสวิชา, ชื่อวิชา, กลุ่มเรียน/Section
- วันที่สอน, เวลาเริ่ม–สิ้นสุด, ห้องเรียน
- ประเภทปัญหา (ลืมเช็กอิน / ระบบขัดข้อง / อินเทอร์เน็ตมีปัญหา / ตารางสอนไม่ตรง / อื่นๆ)
- เหตุผลประกอบ, แนบหลักฐาน
