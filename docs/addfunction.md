# การจัดการ Permission และบทบาทผู้ใช้งาน

## สิทธิ์การมองเห็นเมนูตามบทบาท

### 1. อาจารย์ (`teacher`)
- เช็คอิน / ยื่นคำร้อง
- คำขออนุมัติของฉัน

### 2. คณบดี (`dean`)
- รออนุมัติ
- ประวัติคำขอที่พิจารณาแล้ว

### 3. ผอ.สำนักงานวิชาการ / ผอ.กลุ่มงาน / วิชาการ (`director` / `academic`)
- คำขอที่อนุมัติแล้ว
- อีเมลแจ้งเตือน

### 4. ผู้ดูแลระบบ (`admin`)
- เห็นทุกเมนูของทุกบทบาท
- เพิ่มเติม: ภาพรวมสถิติ, นำเข้า Excel, ข้อมูลหลัก, บันทึกระบบ

## การจัดการสิทธิ์

1. **สิทธิ์เริ่มต้น**: ผู้ใช้ใหม่ที่ login ครั้งแรกจะได้รับสิทธิ์ `teacher` อัตโนมัติ (จัดการโดย `POST /api/users/login` ใน backend)
2. **Admin เปลี่ยนบทบาท**: ผ่านหน้าจอ **ข้อมูลหลัก** → แท็บ **ข้อมูลผู้ใช้งานในระบบ** → Dropdown เลือกบทบาท (เรียก `PATCH /api/users/:email/role`)

## ไฟล์ที่เกี่ยวข้อง

| ไฟล์ | หน้าที่ |
|------|--------|
| `backend/src/routes/users.js` | API จัดการ login และเปลี่ยน role |
| `frontend/src/App.jsx` | ตรวจสอบ role → แสดงเมนูตามสิทธิ์ |
| `frontend/src/components/Sidebar.jsx` | กำหนดเมนูที่แสดงตาม role |
