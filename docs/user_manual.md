# 📖 คู่มือการใช้งานระบบคำร้องขอเช็คอินย้อนหลัง (RSMS User Manual)

ระบบ **RSMS (Request Submission and Management System)** เป็นระบบที่ช่วยให้อาจารย์ผู้สอนยื่นขอเช็คอินการสอนย้อนหลังเมื่อลืมเช็คอินหรือเกิดเหตุขัดข้องทางเทคนิค โดยส่งคำขอไปยังคณบดีเพื่อพิจารณาอนุมัติ และฝ่ายวิชาการสามารถเข้ามาตรวจสอบสรุปสถิติภาพรวมได้ คู่มือฉบับนี้แยกรายละเอียดขั้นตอนการใช้งานตามสิทธิ์ผู้ใช้งาน (Roles) ในระบบ

---

## 🏛️ สรุปบทบาทผู้ใช้งานในระบบ (System Roles Overview)

| บทบาท (Role) | คำอธิบายภาษาไทย | สิทธิ์การเข้าถึงเมนูและการทำงานหลัก |
| :--- | :--- | :--- |
| **อาจารย์ (`teacher`)** | อาจารย์ผู้สอน | ยื่นคำร้องย้อนหลัง, ติดตามสถานะคำร้องของตนเอง |
| **คณบดี (`dean`)** | คณบดีแต่ละคณะ | รับเมลสรุปประจำวัน (20.00 น.), อนุมัติ/ปฏิเสธผ่านเมลหรือในระบบ |
| **ผอ.สำนักงานวิชาการ (`director`)** | ผู้อำนวยการวิชาการ | ดูประวัติคำร้องอนุมัติแล้ว, ดูภาพรวมสถิติรายงานระบบ |
| **ผอ.กลุ่มงาน (`group_director`)** | ผู้อำนวยการกลุ่มงาน | ดูประวัติคำร้องอนุมัติแล้ว, ดูภาพรวมสถิติรายงานระบบ |
| **วิชาการ (`academic`)** | เจ้าหน้าที่สำนักงานวิชาการ | ดูประวัติคำร้องอนุมัติแล้ว, ดูภาพรวมสถิติรายงานระบบ |
| **ผู้ดูแลระบบ (`admin`)** | แอดมินผู้ดูแลระบบ | จัดการตั้งค่าเมล, นำเข้า Excel, จัดการข้อมูลผู้ใช้และข้อมูลหลักทั้งหมด |

---

## 1. 🧑‍🏫 คู่มือสำหรับ: อาจารย์ (Teacher)

บทบาทอาจารย์ใช้สำหรับสร้างคำร้องและตรวจติดตามสถานะคำร้องของตนเอง

### 1.1 การยื่นคำร้องขอเช็คอินย้อนหลัง (Submit a Request)
1. เข้าสู่ระบบ -> เลือกเมนู **"กรอกข้อมูลการ Checkin ย้อนหลัง"**
2. กรอกรายละเอียดข้อมูลการสอนให้ครบถ้วน:
   * **วิชาที่สอน**: เลือกวิชา และระบุกลุ่มเรียน (Section)
   * **วันที่สอน**: เลือกวันที่ที่ลืมเช็คอิน
   * **เวลาสอน**: ระบุช่วงเวลาสอน (เช่น 09.00 - 12.00 น.)
   * **ห้องเรียน**: ระบุหมายเลขห้องเรียน
   * **ประเภทปัญหา**: เลือกปัญหาที่เกิดขึ้น (เช่น *ลืมเช็คอิน, ระบบเครื่องลงทะเบียนขัดข้อง, ขอสลับคาบการสอน*)
   * **เหตุผลการยื่นคำร้อง**: อธิบายรายละเอียดความจำเป็น
   * **หลักฐานแนบ**: ทำการอัปโหลดไฟล์ภาพการสอน หรือไฟล์หลักฐานประกอบอื่นๆ (ถ้ามี)
3. ตรวจสอบข้อมูลทั้งหมดแล้วกดปุ่ม **"ส่งคำร้อง"**
4. ระบบจะส่งคำร้องเข้าสู่ตารางรอพิจารณา และเตรียมส่งเมลแจ้งเตือนไปหาคณบดีประจำคณะของผู้สอนโดยอัตโนมัติ

### 1.2 การเขียนคำร้องจากรายการคาบเรียนที่ไม่ได้เช็คอิน (Submit via Unchecked Classes List)
นอกจากวิธีป้อนข้อมูลเองแล้ว อาจารย์สามารถส่งคำร้องจากประวัติคาบสอนของตนเองที่ไม่มีการเช็คอินในระบบได้โดยตรง:
1. **การดึงข้อมูลเข้าระบบ**: ทุกวันที่ 15 ของเดือน ระบบจะทำการรวบรวมและดึงข้อมูลคาบเรียนทั้งหมดที่อาจารย์ผู้สอนไม่ได้เช็คอินเข้ามาเก็บไว้ในระบบโดยอัตโนมัติ
2. **การยื่นคำร้อง**:
   * ไปที่เมนู **"รายชื่ออาจารย์ที่ไม่เช็คอิน"** (หรือ **"รายชื่ออาจารย์ที่ไม่ได้เช็คอิน"**)
   * หน้าจอจะแสดงรายการคาบเรียนของคุณที่ระบบตรวจพบว่าไม่มีข้อมูลการเช็คชื่อ
   * กดปุ่ม **"เขียนคำร้อง"** ท้ายแถวคาบเรียนที่ต้องการ
   * ระบบจะเปิดแบบฟอร์มยื่นคำร้องพร้อมป้อนข้อมูลรายละเอียดของคาบเรียนดังกล่าว (รหัสวิชา, วันที่สอน, เวลาสอน, ห้องเรียน) ให้โดยอัตโนมัติทันที
   * อาจารย์เพียงแค่กรอกประเภทปัญหา, เขียนระบุเหตุผลความจำเป็น, แนบไฟล์หลักฐานประกอบ และกดส่งคำร้องได้ทันที

### 1.3 การตรวจสอบสถานะคำร้อง (Track My Requests)
1. ไปที่เมนู **"คำขออนุมัติของฉัน"**
2. คุณจะเห็นตารางแสดงสถานะคำร้องทั้งหมดที่เคยยื่น:
   * 🟡 **Pending (รอพิจารณา)**: อยู่ในระหว่างรอคณบดีกดตัดสินใจ
   * 🟢 **Approved (อนุมัติแล้ว)**: คณบดีอนุมัติเรียบร้อยแล้ว
   * 🔴 **Rejected (ปฏิเสธ)**: คำขอถูกปฏิเสธ (สามารถคลิกดูหมายเหตุเหตุผลการปฏิเสธได้)
3. เมื่อคณบดีพิจารณาคำร้องแล้ว คุณจะได้รับอีเมลแจ้งผลลัพธ์โดยอัตโนมัติทันที

## 🏛️ 2. คู่มือสำหรับ: คณบดี (Dean)

บทบาทคณบดีทำหน้าที่พิจารณาคำร้องอนุมัติหรือปฏิเสธคำร้องของอาจารย์ในคณะตนเอง

### 2.1 การอนุมัติผ่านระบบอีเมลสรุปรายวัน (Daily Digest Email Approval)
เพื่ออำนวยความสะดวก คณบดีจะได้รับอีเมลรายงานสรุปคำร้องใหม่ของคณะตนเอง**ทุกวันเวลา 20.00 น.** (หากไม่มีคำร้องค้างจะไม่มีเมลส่งไป) ซึ่งสามารถตัดสินใจได้ทันทีโดยไม่ต้องล็อกอินเข้าระบบ:
* **การอนุมัติรายรายการ**: คลิกปุ่ม **"อนุมัติ"** สีเขียว หรือ **"ปฏิเสธ"** สีแดง ท้ายแถวรายการในตารางของอีเมล
  * *กรณีปฏิเสธ*: ระบบจะเปิดหน้าเว็บให้กรอกหมายเหตุ/เหตุผลที่ปฏิเสธเพื่อแจ้งอาจารย์ผู้สอน
* **การอนุมัติทั้งหมดในครั้งเดียว**: คลิกปุ่ม **"✅ อนุมัติทั้งหมด (X รายการ)"** ในกล่องข้อความสีเขียวส่วนหัวของอีเมล ระบบจะอนุมัติคำร้องรอพิจารณาทุกรายการของคณะนั้นทันที

### 2.2 การอนุมัติผ่านหน้าจอระบบ (Web-based Approval)
1. ล็อกอินเข้าระบบ -> ไปที่เมนู **"รออนุมัติ"** (จะแสดงรายการเฉพาะผู้ยื่นที่เป็นอาจารย์ในคณะที่ท่านดูแล)
2. ตรวจสอบประวัติการสอน เหตุผล และเปิดดูรูปภาพหลักฐานที่แนบมา
3. คลิกปุ่ม **"อนุมัติ"** หรือ **"ปฏิเสธ"** 
4. ประวัติคำร้องที่ดำเนินการแล้วจะย้ายไปที่เมนู **"ประวัติคำขอที่อนุมัติ"** เพื่อให้ตรวจสอบย้อนหลังได้ตลอดเวลา

## 💼 3. คู่มือสำหรับ: ฝ่ายบริหาร / สำนักงานวิชาการ (Director / Academic)

สำหรับตำแหน่ง **ผอ.สำนักงานวิชาการ**, **ผอ.กลุ่มงาน** และ **วิชาการ** เพื่อตรวจสอบภาพรวมของระบบ

### 3.1 การเข้าดูสรุปประวัติคำร้องที่ผ่านการอนุมัติแล้ว
1. ล็อกอินเข้าระบบ -> เลือกเมนู **"คำขอที่อนุมัติเเล้ว"**
2. ระบบจะแสดงตารางคำร้องทั้งหมดของทุกคณะที่คณบดีพิจารณาอนุมัติผ่านเรียบร้อยแล้วเพื่อใช้สำหรับดำเนินการเช็คอินเข้าระบบหลักของมหาวิทยาลัยต่อไป

### 3.2 การตรวจสอบภาพรวมระบบ (System Overview Dashboard)
1. ไปที่เมนู **"ภาพรวม"**
2. ระบบจะแสดงแผนภูมิกราฟสถิติแสดงผลเชิงภาพรวม เช่น:
   * จำนวนคำร้องแยกตามสถานะ
   * สถิติจำนวนคำร้องที่ยื่นมาแยกรายคณะ
   * สัดส่วนประเภทปัญหาที่อาจารย์มักพบเจอเพื่อนำไปใช้วางแผนแก้ไขปรับปรุงเชิงนโยบาย

## 🔑 4. คู่มือสำหรับ: ผู้ดูแลระบบ (Admin)

แอดมินใช้สำหรับควบคุมระบบ จัดการนำเข้าข้อมูล และบริหารจัดการสิทธิ์ของพนักงาน

### 4.1 การนำเข้าข้อมูลรายชื่ออาจารย์ที่ไม่เช็คอิน (Import Data)
1. เตรียมไฟล์ Excel ตามรูปแบบที่ระบบกำหนด
2. ไปที่เมนู **"นำเข้า Excel"** หรือ **"นำเข้าข้อมูลอาจารย์ที่ไม่เช็คอิน"**
3. อัปโหลดไฟล์ Excel เข้าสู่ระบบ เพื่อสร้างข้อมูลประวัติการไม่เช็คอินสำหรับให้อาจารย์ใช้ล็อกอินเข้ามาเขียนคำร้องอ้างอิงย้อนหลัง

### 4.2 การจัดการสิทธิ์และข้อมูลผู้ใช้งาน (User Management)
1. ไปที่เมนู **"ข้อมูลหลัก"** -> เลือกแท็บ **"ข้อมูลผู้ใช้งานในระบบ"**
2. คุณจะเห็นรายชื่อผู้ใช้งานทั้งหมดที่เคยลงทะเบียนเข้าสู่ระบบ
3. **การตั้งหรือเปลี่ยนบทบาทผู้ใช้**:
   * ค้นหาอีเมลผู้ใช้ที่ต้องการ
   * คลิกที่ Dropdown ในแถบ **บทบาท (Role)** 
   * เลือกบทบาทที่ต้องการ (เช่น เปลี่ยนจากอาจารย์ธรรมดาเป็น **คณบดี** หรือ **ผอ.กลุ่มงาน**)
   * *หมายเหตุสำหรับคณบดี*: หากเลือกบทบาทเป็น `dean` (คณบดี) จะต้องทำการเลือก **คณะ (Faculty)** ที่ท่านสังกัดในช่องถัดไปด้วย เพื่อระบบจะได้คัดกรองคำร้องและส่งเมลรายวันไปหาผู้รับที่ถูกต้องครับ
